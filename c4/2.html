使用 DIO 搭建数据访问层

数据访问层总体上使用一个接口提供若干分接口的模式实现, 类似如下风格:

public interface IAPIClient {
    public IActivityClient activityClient { get; }
    public IValueClient valueClient { get; }
}

这样的风格是模仿 Octokit 的 Github Client 来的, 比较实用.

外界使用时只需要制造一个客户端对象即可, 为了让外界使用更有灵活性, 一般不设置为单例.

下面先来介绍一些 DIO 的使用, 大部分是跟着官方文档来的.

下面是 Dio 的基本配置方式:

final options = BaseOptions(
  baseUrl: apiBaseUrl,
  sendTimeout: kSendTimeout,
  receiveTimeout: kReceiveTimeout,
);
_client = Dio(options);

可以通过拦截器来对每个请求进行拦截加工:

Dio _makeDioClient() {
  final options = BaseOptions(
    baseUrl: apiBaseUrl,
    sendTimeout: kSendTimeout,
    receiveTimeout: kReceiveTimeout,
  );
  final client = Dio(options);
  final interceptor = InterceptorsWrapper(
    onRequest: (RequestOptions options) async {
      // TODO: addAll 的时候 other 不能被改动, 这里是否会成为一个缺陷点?
      if (_additionalHeaders != null && _additionalHeaders.isNotEmpty)
        options.headers.addAll(_additionalHeaders);
      return options;
    },
  );
  client.interceptors.add(interceptor);
  return client;
}
